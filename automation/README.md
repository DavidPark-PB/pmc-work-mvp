# PMC-AUTO

한국 도매 상품 → 해외 쇼핑몰(eBay, Shopify, Alibaba, Shopee) 자동 업로드 시스템

## 시스템 개요

PMC-AUTO는 쿠팡 등 한국 온라인쇼핑몰에서 수집한 상품 정보를 CSV 파일로 업로드하면,
가격을 자동 계산하여 해외 플랫폼에 일괄 등록해주는 자동화 도구입니다.

**지원 플랫폼:**

| 플랫폼 | 판매 통화 | 기본 수수료 | 비고 |
| --- | --- | --- | --- |
| eBay | USD | 13% | Trading API (XML) |
| Shopify | USD | 5% | REST Admin API |
| Alibaba | USD | 5% | 국제 B2B |
| Shopee | USD | 6% | 동남아 마켓 |

**팀 협업 지원:** 관리자/팀원 역할 분리, 소유권 기반 작업 분배

## 설치 및 설정

### 요구사항

- Node.js 18+
- PostgreSQL (Supabase 권장)
- Chrome 브라우저 (크롤러용)

### 설치

```bash
git clone https://github.com/inwon100/ccorea-auto.git
cd ccorea-auto
npm install
npx patchright install chrome
```

### DB 초기화 및 서버 실행

## 사용 가이드

### 1. 첫 접속 & 사용자 설정

처음 접속하면 이름 설정 모달이 표시됩니다.

- **팀원:** 이름을 입력하고 "확인"을 누릅니다.
- **관리자:** "관리자로 로그인"을 체크하고 관리자 비밀번호를 입력한 뒤 "확인"을 누릅니다.

> 관리자 비밀번호는 `.env` 파일의 `SETTINGS_PASSWORD`에 설정한 값입니다.

사이드바 하단에서 현재 로그인된 이름과 역할(Admin/팀원)을 확인할 수 있으며,
클릭하면 이름을 변경하거나 관리자 로그인을 다시 할 수 있습니다.

### 2. CSV 업로드 (관리자 전용)

사이드바에서 **"CSV 업로드"**를 클릭하거나, `/upload-csv` 페이지로 이동합니다.

- 파일을 드래그 앤 드롭하거나 클릭하여 선택합니다.
- `.csv` 파일만 허용되며, 최대 50MB까지 업로드 가능합니다.
- 업로드가 완료되면 자동으로 다음 단계로 이동합니다.

### 3. 데이터 미리보기 & DB 등록 (관리자 전용)

업로드된 CSV 데이터를 미리보기 테이블에서 검토합니다.

- 최대 20개 행이 표시되며, 이미지/상품명/가격/링크를 확인할 수 있습니다.
- **"DB에 등록하기"** 버튼을 누르면 데이터가 시스템에 저장됩니다.
- 등록 결과(신규/업데이트/오류 건수)가 표시됩니다.

### 4. 쇼핑몰 선택 & 상품 선택

사이드바에서 **"쇼핑몰 선택"**을 클릭하거나, `/select` 페이지로 이동합니다.

1. **플랫폼 선택:** 상단의 플랫폼 버튼을 클릭하여 업로드할 쇼핑몰을 선택합니다 (복수 선택 가능).
2. **상품 선택:** 체크박스로 업로드할 상품을 선택합니다. 각 상품의 예상 판매가(USD)가 함께 표시됩니다.
3. **업로드 시작:**
   - **"테스트"** 버튼 — 실제 플랫폼에 등록하지 않고 시뮬레이션만 실행합니다.
   - **"업로드 시작"** 버튼 — 선택한 플랫폼에 실제로 리스팅을 등록합니다.

> 자신에게 할당된 상품만 선택할 수 있습니다. 다른 팀원의 상품은 비활성화됩니다.

### 5. 업로드 진행

업로드가 시작되면 실시간 진행률 화면이 표시됩니다.

- 프로그레시브 바로 전체 진행률을 확인할 수 있습니다.
- 각 상품별 성공/실패 결과가 실시간으로 추가됩니다.
- 모든 작업이 완료되면 "결과 보기" 버튼이 활성화됩니다.

### 6. 결과 확인

업로드 완료 후 결과 화면에서 상세 내역을 확인합니다.

- 전체/성공/실패 건수 통계
- 성공한 항목: 플랫폼 리스팅 URL 링크 제공
- 실패한 항목: 에러 메시지 표시

## 주요 기능

### 대시보드

메인 화면(`/`)에서 시스템 전체 현황을 파악합니다.

- **상품 파이프라인:** 업로드 대기 / 판매중 / 판매 종료 탭별 상품 목록
- **진행 중 작업:** 현재 실행 중인 업로드 작업이 있으면 배너로 표시

### 히스토리

`/history` 페이지에서 과거 작업 기록을 확인합니다.

- **세션 잡 기록:** 현재 세션에서 실행한 업로드 작업 목록
- **리스팅 기록:** 생성된 모든 리스팅 (상태별 필터, 페이지네이션)
- **CSV 업로드 이력:** 과거 업로드한 CSV 파일 목록 (미등록 파일은 "등록" 버튼 제공)

### 가격 설정 (관리자 전용)

`/settings` 페이지에서 플랫폼별 가격 전략을 설정합니다.

각 플랫폼(eBay, Shopify, Alibaba, Shopee)별로 다음 항목을 조정할 수 있습니다:

| 설정 항목 | 설명 | 기본값 |
| --- | --- | --- |
| 마진율 (%) | 목표 수익률 | 30% |
| 환율 (KRW/USD) | 원화→달러 환율 | 1,400 |
| 수수료율 (%) | 플랫폼 판매 수수료 | 플랫폼별 상이 |
| 기본 배송비 (원) | 해외 배송 기본 비용 | 5,500원 |

**실시간 환율 조회** 버튼으로 현재 환율을 자동 적용할 수 있습니다.
하단의 가격 미리보기에서 원가를 입력하면 각 플랫폼별 예상 판매가를 즉시 확인할 수 있습니다.

### 휴지통

`/trash` 페이지에서 삭제된 상품을 관리합니다.

- **복원:** 선택한 항목을 다시 활성 상태로 되돌립니다.
- **완전 삭제:** 선택한 항목을 영구적으로 삭제합니다 (복구 불가).

### 재고 동기화

서버가 실행 중일 때 **30분 간격**으로 활성 리스팅의 재고를 플랫폼과 자동 동기화합니다.
히스토리 화면에서 수동 동기화도 실행할 수 있습니다.

## 팀 협업

### 역할 및 권한

| 기능 | 관리자 | 팀원 |
| --- | --- | --- |
| CSV 업로드 | O | X |
| DB 등록 (import) | O | X |
| 상품 분배 | O | X |
| 상품 업로드 | O (자기 할당분) | O (자기 할당분만) |
| 가격 설정 | O | X |

### 작업 흐름

1. **관리자**가 CSV를 업로드하고 DB에 등록합니다.
2. **관리자**가 등록된 상품을 각 팀원에게 분배합니다.
3. **팀원**은 자신에게 할당된 상품만 확인하고 업로드할 수 있습니다.
4. 다른 팀원의 상품은 "OOO 할당"으로 표시되며 선택할 수 없습니다.
5. 미할당 상품은 관리자만 접근할 수 있습니다.

## CSV 형식

CSV 파일은 쿠팡 등 한국 쇼핑몰 크롤 데이터 형식을 따릅니다.

### 컬럼 순서

| 인덱스 | 내용 | 필수 |
| --- | --- | --- |
| 0 | 이미지 URL | O |
| 1 | 상품 URL | O |
| 2 | (예비) | - |
| 3 | 상품명 | O |
| 4 | 가격 (원) | O |
| 5 | 별점 | - |
| 6 | 리뷰 수 | - |
| 7-8 | (예비) | - |
| 9 | 할인율 | - |
| 10 | 원가 (원) | - |

- 첫 번째 행은 헤더로 인식되어 건너뜁니다.
- 가격 필드는 "12,900원" 같은 문자열도 자동으로 숫자로 변환됩니다.
- 큰따옴표(`"`)로 감싼 필드를 지원합니다.

## 가격 계산 공식

```
판매가(USD) = (매입가(KRW)/환율 + 배송비(KRW)/환율) × (1 + 마진율) / (1 - 수수료율)
```

**예시** (eBay 기준, 기본 설정):

- 매입가: 15,000원
- 환율: 1,400
- 배송비: 5,500원
- 마진율: 30%
- 수수료: 13%

```
판매가 = (15000/1400 + 5500/1400) × 1.30 / 0.87
       = (10.71 + 3.93) × 1.30 / 0.87
       = 14.64 × 1.4943
       = $21.88
```

## 트러블슈팅

### eBay 토큰 만료

eBay User Token은 주기적으로 갱신이 필요합니다.
`EBAY_REFRESH_TOKEN`이 유효하면 시스템이 자동으로 Access Token을 갱신합니다.
Refresh Token도 만료된 경우 eBay Developer 콘솔에서 재발급해야 합니다.

### Shopee 토큰 만료

Shopee Access Token은 **4시간**마다 만료됩니다.
시스템이 자동으로 Refresh Token을 사용하여 갱신하지만,
Refresh Token도 만료되면 Shopee Open Platform에서 재인증이 필요합니다.

### CSV 파싱 오류

- 파일이 **UTF-8** 인코딩인지 확인합니다.
- 최소 5개 컬럼이 필요합니다 (이미지, URL, 예비, 상품명, 가격).
- 상품명 또는 URL이 빈 행은 자동으로 건너뜁니다.

### 서버 포트 충돌

기본 포트는 3000입니다. 충돌 시 `.env` 파일에서 `PORT=3099` 등으로 변경할 수 있습니다.
